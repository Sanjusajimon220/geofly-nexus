GeoFly NEXUS

An AI-enhanced 3D geospatial intelligence platform — semantic segmentation and urban analytics from high-resolution aerial imagery.

A single deep-learning model turns raw aerial imagery and LiDAR into a live, browsable urban-intelligence platform: per-pixel land cover, five derived analytical layers, and per-tile reporting across eight themes — deployed to the public web at production scale.

01

The work

GeoFly NEXUS is an end-to-end system, not just a model. A modified SegFormer-B3 transformer performs 7-class land-cover segmentation on a custom 5-channel input — Red, Green, Blue, Near-Infrared, and a normalised Digital Surface Model (nDSM). Its per-pixel predictions feed an analytical pipeline that produces derived intelligence layers and structured per-tile reports, all served through a single-page 3D web viewer.

The work spans the full arc a production data-science role demands: dataset curation and labelling, iterative model development with error-driven relabelling, a reproducible Python processing pipeline, and a live cloud deployment validated across two independent regions of Germany.

02

Approach

The model was developed iteratively. Each of four training rounds (v1→v4) used error analysis to expose a specific failure mode, which was then addressed through targeted relabelling and class-weight adjustment — producing monotonic improvement rather than blind hyperparameter search.

// ARCHITECTURE

5-channel SegFormer-B3

Transformer backbone adapted from 3-channel RGB to a 5-channel RGB+NIR+nDSM input, fusing spectral and elevation cues for sharper class boundaries.

// DATA PIPELINE

Imagery + LiDAR fusion

Python chain over GDAL / Rasterio / GeoPandas / laspy: DSM/DTM/nDSM generation, multispectral fusion, tiling, batch inference and TMS tile serving — config-driven and reproducible.

// METHOD

Error-driven iteration

Four rounds, each fixing a diagnosed failure mode. Experiment tracking, YAML configuration, Docker and Git workflows throughout.

03

Results

The deployed v4 model reaches 89.42% mIoU and 96.11% pixel accuracy on the held-out evaluation set. Six of the seven foreground classes exceed 86% IoU; the seventh (tree) sits at 72.4%, limited by genuine canopy-boundary ambiguity in 3 cm orthophotos.

04

The platform

Segmentation is only the engine. Predictions drive five derived intelligence layers and a per-tile analytics panel, all rendered as interactive TMS tile pyramids from regional overview (zoom 14) down to building detail (zoom 19).

5 layers: building height NDVI vegetation solar suitability surface sealing flood susceptibility

A per-tile intelligence engine computes structured analytics across eight themes — land cover, building morphology, BauNVO regulatory compliance, environmental & ecological metrics, asset valuation, energy analysis, hydrology, and scenario modelling — and a BLIP vision-language model adds a natural-language scene caption per tile.

05

Production & scale

Multi-region capability was proven, not claimed: the same v4 checkpoint was deployed across a second region — Saarland, 297.65 km² and 82,681 tiles — with no retraining and no code changes. Per-region setup is a single YAML file (coordinate system, LiDAR spec, DSM resolution, optional auxiliary nDSM encoding).

The deployment surfaced two general engineering fixes: a viz150 auxiliary nDSM encoding for terrain exceeding the model’s 30 m input ceiling, and a per-tile morphology safeguard that stopped mining spoil heaps (Bergehalden) being misread as tall buildings.
